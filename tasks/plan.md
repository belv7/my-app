- 検証: `prisma migrate deploy`、`npm start`。
- 依存: なし。

### Task 2: 現行 CRUD の実ブラウザ確認 ✅

- 受け入れ条件: タスクの追加・編集・完了切替・複製・削除が画面から行え、ブラウザのコンソールにエラーがない。
- 検証: localhost で操作し、DOM・コンソール・画面を確認する。
- 依存: Task 1。

## Phase 2: モバイル優先のドット絵 UI

### Task 3: レイアウト基盤をモバイル優先に変更

- 受け入れ条件: 320px では1カラム、768px以上で必要に応じて2カラムとなり、全操作要素が44px以上。
- 検証: 320px / 768px / 1024px / 1440px の画面確認とキーボード操作。
- 依存: Checkpoint A。

### Task 4: 街の表現をドット絵に置換

- 受け入れ条件: 建物・住民・自然物・資源がピクセルアートで表現され、完了数に応じて増える。タスク情報の可読性は落とさない。
- 検証: 完了数 0・1・3・6 の状態で画面確認。
- 依存: Task 3。

### Task 5: 完了時フィードバックを追加

- 受け入れ条件: 完了操作で控えめな視覚フィードバックがあり、`prefers-reduced-motion` では最小化される。
- 検証: 通常設定／動きを減らす設定で確認。
- 依存: Task 4。

### Checkpoint B: UI 承認

- [x] 全ブレークポイントでレイアウトを確認
