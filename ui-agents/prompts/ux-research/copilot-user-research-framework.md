# User Research & Persona Framework for GitHub Copilot

**Category:** UX Research  
**Difficulty:** Intermediate  
**Tags:** #ux-research #personas #user-journey #data-driven #copilot  
**Platform:** GitHub Copilot

## Description

This prompt helps you develop data-driven user personas, map user journeys, and establish research frameworks for your products. It provides structured approaches to understanding your users.

## Prompt

```
You are a UX researcher and persona development expert for GitHub Copilot. Help me create user research frameworks.

Project Context:
- Product/service name:
- Product type (web app, mobile app, SaaS, e-commerce, etc.):
- Current stage (concept, MVP, scaling, mature):
- Existing user data (if any):
- Research timeline and budget:
- Key stakeholders/team members:

Please provide a comprehensive user research framework:

1. **Research Strategy**:
   - Research methods (surveys, interviews, analytics, usability tests, etc.)
   - Sample size and selection criteria
   - Data collection timeline
   - Analysis approach
   - Synthesis methodology

2. **User Personas** (3-5 personas):
   For each persona, provide:
   - Persona name and demographic
   - Background and professional context
   - Goals and motivations
   - Pain points and frustrations
   - Tech proficiency level
   - Device and platform preferences
   - Behavioral patterns
   - Quote/representative statement

3. **User Journey Maps**:
   - Journey stages (awareness, consideration, decision, usage, advocacy)
   - Touchpoints at each stage
   - User emotions and pain points
   - Opportunities for improvement
   - Key moments of truth

4. **User Needs Analysis**:
   - Functional needs (what users need to accomplish)
   - Emotional needs (how they want to feel)
   - Social needs (how they want to be perceived)
   - Unmet needs and opportunities

5. **Segmentation Strategy**:
   - Primary segments based on behavior/needs
   - Segment sizes and characteristics
   - Segment-specific strategies
   - Prioritization criteria

6. **Research Artifacts**:
   - Interview guide template
   - Survey template
   - Usability test script
   - Data collection template
   - Synthesis template

7. **Implementation Roadmap**:
   - Phase 1: Initial research
   - Phase 2: Deep dives
   - Phase 3: Continuous monitoring
   - Validation mechanisms
   - Update frequency

8. **Documentation**:
   - Persona cards template
   - Journey map template
   - Research findings summary
   - Key insights document
   - Recommendations by persona

Format all deliverables as practical, implementation-ready documents.
```

## Example Usage

**In VS Code with Copilot Chat:**

```
@copilot User Research & Persona Framework
Help me develop user personas for our fitness app:
- Product: FitTrack - Workout tracking and progress app
- Type: Mobile app (iOS/Android)
- Current stage: MVP with 5K users
- Timeline: 2-week research sprint
- Key stakeholders: Product manager, design lead, 2 engineers
```

## Sample Results

The agent will provide:
- 3-5 detailed user personas with demographics
- Journey maps for each persona
- Functional, emotional, and social needs analysis
- Research methodology and timeline
- Interview and survey templates
- Usability test scripts
- Data synthesis approach
- User segmentation strategy
- Opportunity identification document
- Persona cards and posters
- Key insights and recommendations
- Continuous research plan

## Tips

- Provide any existing user data or analytics
- Specify your product type and stage
- Request persona variations (heavy user, light user, non-user, etc.)
- Ask for journey mapping for specific use cases
- Request research tools recommendations
- Ask for analysis frameworks (Jobs to be done, etc.)
- Request competitive user analysis
