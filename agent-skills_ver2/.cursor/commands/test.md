# /test — テスト駆動開発

**必読:** `skills/test-driven-development/SKILL.md` を全文読み、手順を省略せず実行する。

## 新機能

1. 期待挙動を記述するテストを書く（**失敗すること**）
2. テストを通す実装を書く
3. テストがグリーンのままリファクタする

## バグ修正（Prove-It パターン）

1. バグを再現するテストを書く（**失敗すること**）
2. テストが失敗することを確認する
3. 修正を実装する
4. テストが通ることを確認する
5. 全テストスイートでリグレッションを確認する

## ブラウザ関連

ランタイム確認が必要な場合は `skills/browser-testing-with-devtools/SKILL.md` も適用する。Cursor のブラウザ MCP が使える場合は DevTools で検証する。

## ユーザー追加入力

コマンド名の後のテキスト（`$ARGUMENTS`）があれば、テスト対象のスコープ・ファイル・バグ説明として扱う。

## 完了条件

- 関連テストがすべてパスしている
- 「たぶん動く」ではなく、実行結果の証拠がある
