# /spec — 仕様駆動開発

**必読:** `skills/spec-driven-development/SKILL.md` を全文読み、手順を省略せず実行する。

## 手順

1. ユーザーが作りたいものを理解する。不明点があれば質問する:
   - 目的とターゲットユーザー
   - コア機能と受け入れ基準
   - 技術スタックの希望と制約
   - 境界（常にやること / 先に確認すること / 絶対にやらないこと）
2. 次の6領域をカバーする構造化仕様を書く:
   - 目的（Objective）
   - コマンド（Commands）
   - プロジェクト構造（Structure）
   - コードスタイル（Code style）
   - テスト戦略（Testing）
   - 境界（Boundaries）
3. 仕様をプロジェクトルートの `SPEC.md` に保存する
4. 実装に進む前にユーザーに確認を取る

## ユーザー追加入力

コマンド名の後に書かれたテキスト（`$ARGUMENTS`）があれば、仕様の初期コンテキストとして扱う。

## 完了条件

- `SPEC.md` が存在し、6領域すべてをカバーしている
- 仮定が明示されている
- ユーザーの確認を得ている（または「この前提で進めて」と明示された）
