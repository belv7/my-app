# /code-simplify — コード簡素化

**必読:** `skills/code-simplification/SKILL.md` を全文読み、手順を省略せず実行する。

## 対象

直近の変更、またはユーザーが指定したスコープ（`$ARGUMENTS`）。

## 手順

1. プロジェクトの規約（`AGENTS.md`、既存コード）を確認する
2. 対象コードの目的、呼び出し元、エッジケース、テストカバレッジを理解する
3. 簡素化の機会を探す:
   - 深いネスト → ガード節やヘルパー抽出
   - 長い関数 → 責務で分割
   - ネストした三項演算子 → if/else
   - 汎用名 → 説明的な名前
   - 重複ロジック → 共通化
   - デッドコード → 確認後に削除
4. **1変更ずつ** 適用し、都度テストを実行する
5. 全テストパス、ビルド成功、差分が意図どおりであることを確認する

## ルール

- **挙動は変えない**。リファクタのみ
- テストが失敗したらその変更を戻し、別案を検討する
- 完了後、必要なら `/review` 相当で品質を確認する
