# /plan — タスク分解

**必読:** `skills/planning-and-task-breakdown/SKILL.md` を全文読み、手順を省略せず実行する。

## 前提

- `SPEC.md`（または同等の仕様）が存在すること。なければ先に `/spec` を実行するよう伝えて停止する。

## 手順

1. **プランモード** — 読み取りのみ。コード変更はしない
2. 既存の仕様と関連コードを読む
3. コンポーネント間の依存関係を特定する
4. **縦スライス**で分解する（水平レイヤー単位ではなく、1タスクで完結する経路）
5. 各タスクに受け入れ基準と検証手順を書く
6. フェーズ間のチェックポイントを追加する
7. 人間のレビュー用にプランを提示する

## 成果物

- `tasks/plan.md` — 依存関係付きの実装プラン
- `tasks/todo.md` — チェックリスト形式のタスクリスト

## ユーザー追加入力

コマンド名の後に書かれたテキスト（`$ARGUMENTS`）があれば、スコープや優先度のヒントとして使う。
